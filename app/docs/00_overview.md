# About_us
    url: /about_us
    состав блоков: текст, изображения, категории. блоки с людьми. у каждого блока здесь отображаются изображние, имя, категория. есть сортировка по этим категориям. 
    данные от бекенда: почти статик страница. изображения и категории берутся из бд.
    действия юзера: может пролистать блоки, отсортировать, кликнуть на блок и перейти на детальную страницу этого блока

# Projects   
    url: /projects
    состав блоков: блоки с проектами. у каждого блока на этой странице есть фоновое изображение(бд), название, краткое описание, кнопка see project которая ведет на project detail

    данные от бекенда: все данные для блоков берутся из бд
    
    действия юзера: может просматривать блоки с проектами, листать с помощью пагинации(но это делает фронт, не я). очень детальная сортировка по типам проекта(resindetial, commercial, public, enterior, exterior, возможно добавление кастомных при создании проекта), по их стилю(all, modern, minimalist, classic, scandinavian, возможно добавление кастомных при создании проекта), по локации(во фронте это выпадающий список с локациями), по времени(в макете это год). есть сортировка общей страницы по дате, поиск по именам проекта(возможно по содержимому) 

# Project detail
    url: /project_detail
    состав блоков: изображения, текст.
    данные от бекенда: данные о проекте(поля модели), изображения в том числе
    действия юзера: может просмотреть проект детально, ознакомиться с ведущими людьми этого проекта, пролистать галерею, также у каждого проекта отдельная галерея с чертежами(опционально). 
    доп заметки по странице: в верхней части страницы название проекта(бд), лого/изображение(превью, тоже из бд), рядом блок с данными (location[specify country/city], completion[specify year], area[~320m2], client[client name], year[on going 2022]), ниже текст(full_text), после него галерея проекта, еще ниже отдельная галерея проекта именно с чертежами. в конце проекта (почти конец страницы, не считая футтера), доп данные по проекту(architect[david morgan], lead designer[elena park], interior design[lucas bennet], project manager[michael chen]). 
    исходя из этого можно понять, какие поля и связи должны быть у модели projects. 

# careers

# contact form
    url: /contact_form
    состав блоков: формы для заполнения своих данных(name ,surname, email, phone number), также форма для отправки портфолио(файл аплоудер)
    данные от бекенда: все эти вводимые данные должны приниматься бекендом
    действия юзера: может заполнить свои данные для той или иной вакансии, отправить свое портфолио pdf/image файлом. 


# contact us (with arhea)

# news & updates
